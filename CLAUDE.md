# 放置帝国 — 文字挂机RTS游戏

## 项目概览

- 类型：文字挂机 RTS，回合制战斗
- 平台：Web（HTML/CSS/JS），单文件架构，安卓 WebView 运行
- 存档：localStorage（key: `rts_save`）
- 当前阶段：Phase 1 — 最小可玩版本

## 文件结构

```
index.html          # 主文件（~1500行），包含全部 HTML/CSS/JS
设计框架.md          # 完整设计文档
设计框架-脑图.html    # 脑图可视化
开发日记.md          # 开发记录
```

### 核心全局对象

- `CFG` — 全部静态配置数据（资源、建筑、兵种、敌人、克制倍率）
- `S` — 运行时状态（资源数量、建筑等级、兵力池、编队、日志）
- `B` — 战斗临时状态（每场战斗独立）

### 数据流

```
CFG(静态配置) + S(存档状态) → UI渲染
S.res / S.buildings / S.pool / S.formation → 操作函数 → S 更新 → 重渲染
S + 敌人配置 → 战斗系统 → B 战斗结果 → S 更新(功勋/掉落/日志)
localStorage ←→ S(序列化)
```

## 开发约定

- 纯原生 JS，不引入任何框架或库
- 所有数据配置化：新增资源/兵种/建筑只需在 `CFG` 添加条目
- CSS 使用手机框架模拟（max-width: 430px）
- 命名：驼峰命名，配置键名与设计文档一致
- 修改 `CFG` 结构时同步更新所有渲染和计算函数
- 所有 UI 文本使用中文

## 设计要点

- 克制倍率：优势 1.3/1.5，劣势 0.7，普通 1.0
- 法师互易伤：双向 1.3×
- 弓兵对非骑兵有 20% miss，对骑兵 50% miss
- 伤害公式：攻击 × 克制 × 易伤 × 随机(0.8-1.2) / 防御
- 建筑升级时间封顶 120 秒
- 资源建筑上限 Lv.10（第一大关）
