# 放置帝国 — 文字挂机RTS游戏

## 项目概述

- 类型：文字挂机 RTS，回合制战斗
- 平台：Web（HTML/CSS/JS），单文件架构，安卓 WebView 运行
- 存档：localStorage（key: `rts_save`）
- 当前阶段：Phase 1 — 最小可玩版本
- 技术栈：原生 ES6，无框架依赖，JSON 配置化

## 文件结构

- `index.html` — 全部代码（HTML/CSS/JS，~1500行）
- `设计框架.md` — 完整设计文档（资源/兵种/战斗/英雄/远征/周目）
- `设计框架-脑图.html` — 脑图可视化
- `开发日记.md` — 开发记录
- `.claude/settings.json` — 团队共享配置（提交 git）
- `.claude/settings.local.json` — 个人配置（不提交 git）

### 核心全局对象

- `CFG` — 全部静态配置数据（资源、建筑、兵种、敌人、克制倍率）
- `S` — 运行时状态（资源数量、建筑等级、兵力池、编队、日志）
- `B` — 战斗临时状态（每场战斗独立）

### 数据流

```
CFG(静态配置) + S(存档状态) → UI渲染
S.res / S.buildings / S.pool / S.formation → 操作函数 → S 更新 → 重渲染
S + 敌人配置 → 战斗系统 → B 战斗结果 → S 更新(功勋/掉落/日志)
localStorage ←→ S(序列化)
```

## 开发规范

1. 所有修改集中在 `index.html` 一个文件
2. 新增配置放到 `CFG` 对象中（数据配置化）
3. 状态管理统一使用 `S` 对象，战斗逻辑用 `B` 状态机
4. 保持手机框体 + 底部导航的 UI 布局
5. 命名：驼峰命名，配置键名与设计文档一致
6. 所有 UI 文本使用中文
7. 提交前确认存档兼容性（localStorage 结构变化需做迁移）
8. 更新 `开发日记.md` 记录新功能

## 设计要点

- 克制倍率：优势 1.3/1.5，劣势 0.7，普通 1.0
- 法师互易伤：双向 1.3×
- 弓兵对非骑兵有 20% miss，对骑兵 50% miss
- 伤害公式：攻击 × 克制 × 易伤 × 随机(0.8-1.2) / 防御
- 建筑升级时间封顶 120 秒，资源建筑上限 Lv.10（第一大关）
