# 文字挂机RTS — 团队规范

## 项目概述

单文件 HTML/CSS/JS 文字挂机 RTS 游戏。无框架依赖，所有逻辑在 `index.html` 一个文件中。

## 技术栈

- 原生 HTML/CSS/JS（ES6）
- JSON 配置化：`CFG` 对象包含所有兵种/建筑/资源/敌人数据
- localStorage 存档
- 手机框体 UI + 底部 5 标签页导航

## 核心架构

| 变量 | 用途 |
|------|------|
| `S` | 主状态对象（资源、建筑等级、军营、编队） |
| `B` | 战斗状态机（独立，不影响主状态） |
| `CFG` | 所有配置数据 |
| `tickTimer` | 1 秒定时器驱动资源产出 + 建筑计时 |

## 目录结构

- `index.html` — 全部代码（HTML/CSS/JS）
- `开发日记.md` — 功能规划和进度
- `设计框架.md` — 设计文档
- `.claude/settings.json` — 团队共享配置（可提交 git）
- `.claude/settings.local.json` — 个人配置（不提交 git）

## 团队协作模式

- 本项目的 Agent Teams 模式已启用
- 使用 `@member-name` 给团队成员分配任务
- 任务通过任务面板跟踪（`Ctrl+T`）
- 团队成员各自有独立上下文，适合并行开发不同模块
- 如有文件冲突风险，使用 `claude --worktree` 隔离

## 开发规范

1. 所有修改集中在 `index.html` 一个文件
2. 新增配置放到 `CFG` 对象中
3. 状态管理统一使用 `S` 对象
4. 战斗逻辑放到 `B` 状态机
5. 保持手机框体 + 底部导航的 UI 布局
6. 更新 `开发日记.md` 记录新功能
7. 提交前确认存档兼容性（localStorage 结构变化需做迁移）
