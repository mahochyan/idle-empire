# 放置帝国 — 文字挂机RTS游戏

## 项目概述

- 类型：文字挂机 RTS，回合制战斗
- 平台：Web（HTML/CSS/JS），单文件架构，安卓 WebView 运行
- 存档：localStorage（key: `rts_save`）
- 当前阶段：Phase 1 — 最小可玩版本
- 技术栈：原生 ES6，无框架依赖，JSON 配置化

## 文件结构

- `index.html` — 全部代码（HTML/CSS/JS，~1500行）
- `设计框架.md` — 完整设计文档（资源/兵种/战斗/英雄/远征/周目）
- `设计框架-脑图.html` — 脑图可视化
- `开发日记.md` — 开发记录
- `.claude/settings.json` — 团队共享配置（提交 git）
- `.claude/settings.local.json` — 个人配置（不提交 git）

### 核心全局对象

- `CFG` — 全部静态配置数据（资源、建筑、兵种、敌人、城镇等级表、克制倍率）
- `S` — 运行时状态（资源、建筑等级、人口分配、兵力池、编队、日志）
- `B` — 战斗临时状态（每场战斗独立）

### 核心机制：人口分配采集

```
城镇等级 → 人口上限
人口分配 → 各资源采集量 = 分配人口 × 0.5/秒 × (1 + 建筑Buff%)
建筑升级 → 提升对应资源Buff%
击败Boss → 解锁城镇升级(前期每关1次，后期需跨多关)
```

### 数据流

```
CFG(配置) + S(状态) → UI渲染
S.popAlloc × basePerPop × (1 + buildingBuff) → S.res 自动增长
S + 敌人配置 → 战斗系统 → B 战斗结果 → S 更新(功勋/掉落/Boss计数)
S.townLv / S.popAlloc / S.buildings / S.res → localStorage 存档
```

## 开发规范

1. 所有修改集中在 `index.html` 一个文件
2. 新增配置放到 `CFG` 对象中（数据配置化）
3. 状态管理统一使用 `S` 对象，战斗逻辑用 `B` 状态机
4. 保持手机框体 + 底部导航的 UI 布局
5. 命名：驼峰命名，配置键名与设计文档一致
6. 所有 UI 文本使用中文
7. 提交前确认存档兼容性（localStorage 结构变化需做迁移）
8. 更新 `开发日记.md` 记录新功能

## 设计要点

- 资源公式：`产出 = 分配人口 × 0.5 × (1 + 建筑Buff)`，建筑不再直接产出
- 城镇升级：击败Boss数达标后可升，前期每关1次，后期需跨多关
- 建筑Buff曲线：base + lv×perLv（例如伐木场 Lv.1: +18%, Lv.10: +90%）
- 人口上限由城镇等级决定（Lv.1小村庄=10人 → Lv.8城市=140人）
