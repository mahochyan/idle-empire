# 放置帝国 (Idle Empire)

文字挂机 RTS — 人口分配采集 × 军粮维持 × 回合制自动战斗

## 玩法

**采集** — 分配人口到木材/石料/食物，建筑提供效率Buff，挂机自动产出。

**建军** — 训练五种兵种（步/弓/骑/枪/法），克制闭环 + 兵种被动 + 法师独立互易伤。所有士兵消耗食物维护，军队规模受经济制约。

**出战** — 前/中/后排编队，每格即为一个兵团（人数=HP）。击败Boss后阵容槽位扩展。

**战斗** — 回合制自动演算，速度决定出手顺序，前排保护后排，远程60%被前排阻挡。伤害 = 兵团人数 × ATK / DEF × 克制 × 易伤 × 随机。战斗过程文字播报 + CSS像素动画。

**成长** — 击败Boss → 解锁新兵种 + 城镇可升级 + 建筑可升级 → 人口上限提升 → 更强军队 → 挑战更强敌人。

## 技术框架

4 文件模块化，无框架依赖：

| 文件 | 职责 |
|------|------|
| `index.html` | DOM骨架 + 完整CSS（像素风暗色皮肤、战斗动画、响应式） |
| `sprites.js` | 32×32像素图标集 + `pix()` 渲染函数 |
| `math.js` | CFG配置 + S状态 + 全部游戏逻辑（tick/队列/战斗/存档） |
| `ui.js` | HTML拼接渲染 + DOM事件 + 导航 + toast |

加载顺序：sprites.js → math.js → ui.js

核心对象：
- `CFG` — 静态配置（资源/建筑/兵种/敌人/城镇/克制表）
- `S` — 运行时状态（资源/人口/兵力/编队/队列/日志）
- `B` — 战斗临时状态（每场独立，单位数组/回合/消息）

1秒tick驱动：资源产出 → 建筑计时 → 队列消费 → UI刷新。战斗独立异步循环。

部署：GitHub Pages (`mahochyan.github.io/idle-empire`)，存档：localStorage。

## 当前实现

| 模块 | 内容 |
|------|------|
| 城镇 | 10级（小村庄→城市），Boss驱动升级 |
| 人口 | 分配工人到资源采集，实时调节比例 |
| 建筑 | 伐木场/采石场/农田(产出Buff)、步兵营/靶场/兽栏/枪兵墓穴/法师塔(训练)、营帐(出战上限)、仓库(存储上限) |
| 资源 | 木材/石料/食物，产出 = 人口 × 0.5/秒 × (1 + Buff%) |
| 兵种 | 步弓(初始) → 骑(Boss1) → 枪+法(Boss2)，每人消耗食物维护费 |
| 被动 | 步兵ATK+10%、骑兵闪避10%、枪兵暴击10%(2×)、弓兵基础MISS 20%(对骑50%)、法师互易伤1.3× |
| 训练 | 排队生产系统，资源不足/后备满时暂停，队列上限=后备×10 |
| 编队 | 前/中/后排，初始1×3起步，击败Boss扩展槽位 |
| 战斗 | 回合制，速度排序，近战仅前排攻击，远程60%前排阻挡，动画(弹跳/震动/淡出) |
| 敌人 | 3章×9关（6普通+3Boss），分段解锁对应兵种 |
| UI | 手机框体400px + 5标签页（主页/建筑/军营/战斗/日志） |
