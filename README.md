# 放置帝国 (Idle Empire)

文字挂机 RTS 游戏 — 人口分配采集 × 军备养成 × 回合制自动战斗

## 当前开发状态

**Phase 1 完成（大重构）**

### 核心机制

```
城镇等级 → 人口上限 → 人口分配到资源采集
    ↓
建筑升级(Buff%) → 采集效率提升
    ↓
训练军队 → 侦察敌人 → 阵容编排 → 自动战斗
    ↓
击败 Boss → 解锁新兵种 + 城镇可升级 + 建筑可升级
```

### 功能一览

| 模块 | 内容 |
|------|------|
| 城镇系统 | 8级（小村庄→城市），Boss驱动升级，人口10→140 |
| 人口分配 | 自由调配工人到木材/石料/食物，实时调节 |
| 资源采集 | 分配人口 × 0.5/秒 × (1 + 建筑Buff%) |
| 建筑Buff | 伐木场/采石场/农田提供效率倍率，每级+20% |
| 兵种系统 | 步兵/弓兵(初始) → 骑兵(Boss1) → 枪兵+法师(Boss2) |
| 克制闭环 | 步克枪→枪克骑→骑克弓→弓克步，法师独立互易伤 |
| 阵型系统 | 前/中/后排，兵团槽位，人数=HP |
| 战斗系统 | 回合制自动演算，文字逐行播报，CSS动画 |
| 敌人关卡 | 3章共9关（6普通+3Boss），分段难度递增 |
| UI | 手机框体 + 5标签页导航 |
| 存档 | localStorage，兼容旧版存档自动迁移 |

## 技术栈

- 原生 HTML/CSS/JS（ES6），单文件 `index.html`（~1470行）
- JSON配置化：`CFG`对象包含所有数据
- 1秒tick驱动 + 战斗独立状态机`B`
- 部署：GitHub Pages (`mahochyan.github.io/idle-empire`)

## 核心数据流

```
CFG(配置) + S(状态) → UI渲染
S.popAlloc × 0.5 × (1 + buildingBuff) → S.res 增长
S + 敌人CFG → 战斗系统 → B结果 → S更新
localStorage ←→ S序列化存档
```

## 伤害公式

```
伤害 = 攻击 × 克制倍率 × 法师易伤 × 随机浮动(0.8~1.2) − 防御减免
```

- 克制倍率：优势1.3~1.5 / 普通1.0 / 劣势0.7
- 弓兵基础20%miss，对骑兵50%miss
- 法师互易伤（双向）：1.3×
