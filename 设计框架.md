# 文字挂机RTS游戏 - 设计框架 v3

---

## 一、游戏概述

- 类型：文字挂机 RTS，回合制战斗
- 平台：Web 技术（HTML/CSS/JS），安卓浏览器/WebView 运行
- 存档：localStorage 本地存储
- 风格：资源驱动军备，RTS 指挥感，纯文字界面

### 核心循环

```
城镇等级 → 人口上限 → 人口分配采集资源
    ↓
建筑升级(效率Buff) → 资源产出加速
    ↓
训练军队 → 侦察敌人 → 调整阵容 → 自动战斗
    ↓
获得功勋/掉落 → 击败Boss → 城镇可升级 → 下一大关
```

---

## 二、城镇与人口系统（核心大本营）

### 2.1 城镇等级

城镇是整个游戏的大本营，决定人口上限。击败守关 Boss 后可升级。

| 城镇等级 | 名称 | 人口上限 | 升级条件 |
|----------|------|---------|----------|
| Lv.1 | 小村庄 | 10 | 初始 |
| Lv.2 | 村庄 | 18 | 击败第 1 大关 Boss |
| Lv.3 | 大村庄 | 28 | 击败第 2 大关 Boss |
| Lv.4 | 小镇 | 42 | 击败第 3 大关 Boss |
| Lv.5 | 城镇 | 60 | 击败第 5 大关 Boss（需跨 2 关） |
| Lv.6 | 大城镇 | 80 | 击败第 7 大关 Boss（需跨 2 关） |
| Lv.7 | 小城 | 105 | 击败第 10 大关 Boss（需跨 3 关） |
| Lv.8 | 城市 | 140 | 击败第 13 大关 Boss（需跨 3 关） |

> 前期每关可升一级，后期需要跨越多关才能升级，形成长线目标。

### 2.2 人口分配采集

- 人口是资源采集的唯一来源，不是建筑
- 玩家将人口分配到各资源类型上工作
- 每人每秒基础产出固定（如 0.5/秒），建筑效率倍率加成
- 可随时调整人口分配比例
- 未分配人口不产生任何资源

## 三、资源系统

### 3.1 分层资源

| 层级 | 资源 | 解锁前提 | 解锁内容 |
|------|------|----------|----------|
| T1 | 木材、石料、食物 | 初始可用 | 基础兵种、初级建筑 |
| T2 | 铁矿、煤炭 | 矿场 N 级 | 中级兵种、阵型扩展 |
| T3 | 秘银、水晶 | 秘法建筑 N 级 | 高级兵种、法师解锁 |
| T4 | 龙晶、神血 | 特殊建筑 N 级 | 神级兵种 |

### 3.2 采集公式

```
实际产出/秒 = 分配人口 × 基础采集率(0.5/秒) × (1 + 建筑效率Buff)
```

- 分配人口：玩家在该资源上投入的劳动力数量
- 基础采集率：每人每秒产出基准值，所有资源类型相同
- 建筑效率Buff：对应采集建筑等级提供的倍率加成

示例：
```
伐木场 Lv.3（Buff +40%），分配 5 人口到木材
木材产出 = 5 × 0.5 × 1.4 = 3.5/秒
```

### 3.3 建筑角色变化

建筑不再直接产出资源，改为提供效率倍率：

| 建筑 | 作用 | Buff 曲线（Lv.1→Lv.10） |
|------|------|------------------------|
| 伐木场 | 木材采集效率 | +10% → +200% |
| 采石场 | 石料采集效率 | +10% → +200% |
| 农田 | 食物采集效率 | +10% → +200% |
| 矿场 | 铁矿采集效率 | +10% → +200% |

- 升级建筑提升效率倍率
- 建筑之间有依赖关系（T2 建筑需消耗 T1 资源建造）

## 四、兵种系统

### 4.1 兵种体系总览

```
                    ┌── 步兵 ──→ 枪兵
普通兵种（4类）    │    ↑         ↓
 克制闭环          │    │         │
                    │  弓兵 ←── 骑兵
                    └──────────────────┐
                                       │
法师（独立体系）                        │
 对所有普通兵：互易伤                   │
 法师打普通兵 → 伤害加成               │
 普通兵打法师 → 伤害加成               │
```

### 4.2 四基础普通兵种 + 克制闭环

| 编号 | 名称 | 种族 | 定位 | 攻击类型 |
|------|------|------|------|----------|
| A | 步兵 | 人族 | 前排/均衡/格挡 | 物理近战 |
| B | 弓兵 | 精灵 | 后排/远程/暴击 | 物理远程 |
| C | 骑兵 | 兽人 | 前排/突击/吸血 | 物理冲锋 |
| D | 枪兵 | 亡灵 | 中排/反骑/破甲 | 穿刺伤害 |

**克制闭环（经典 RTS）：**

```
步兵 ──克──→ 枪兵（步兵近身，枪兵长兵不便近战）
  ↑              ↓
  │              │
弓兵 ←──克── 骑兵（弓兵远程风筝骑兵）
```

- **步兵 克 枪兵**：贴身近战，长枪施展不开
- **枪兵 克 骑兵**：长枪反冲锋，克制骑乘单位
- **骑兵 克 弓兵**：高速冲锋切入后排
- **弓兵 克 步兵**：远程消耗，步兵靠近前已受损

### 4.3 法师 — 独立体系

法师不在四兵种克制环内，有独立的互易伤机制：

| 特性 | 说明 |
|------|------|
| 法师 → 普通兵 | 伤害加成 1.3×（法术穿透物理防御） |
| 普通兵 → 法师 | 伤害加成 1.3×（法师脆皮，近身即死） |
| 法师 ↔ 法师 | 正常伤害 1.0× |

> 定位：玻璃大炮。高输出但极度脆弱，需要前排保护。
> 普通兵打法师也痛，所以对方有法师时，你的骑兵切后排收益极高。

### 4.4 种族被动标签

种族作为被动特性挂在兵种上：

| 兵种 | 种族 | 被动增益 |
|------|------|----------|
| 步兵 | 人族 | 格挡率 +8% |
| 弓兵 | 精灵 | 暴击率 +10% |
| 骑兵 | 兽人 | 吸血率 +8% |
| 枪兵 | 亡灵 | 破甲率 +10% |
| 法师 | 亡灵 | 法术伤害 +12% |

### 4.5 升级线（分期实现）

| 阶段 | 内容 |
|------|------|
| Phase 1 | 4 基础普通兵种 + 法师基础形态 |
| Phase 2 | 每个兵种 1 条升级线（基础→进阶→精锐） |
| Phase 3 | 每个兵种分支（攻击/防御路线） |
| Phase 4 | 神级单位（双分支） |

神级分支设计（预留）：
- 步兵 → 神·圣殿骑士（攻）/ 神·铁壁守卫（防）
- 弓兵 → 神·风行者（攻）/ 神·猎命者（防）
- 骑兵 → 神·毁灭骑士（攻）/ 神·不朽铁骑（防）
- 枪兵 → 神·龙枪将（攻）/ 神·不屈之壁（防）
- 法师 → 神·大魔导（攻）/ 神·死灵王（防）

## 五、战斗系统

### 5.1 基本规则

- 回合制，自动演算
- 阵型系统：前排/中排/后排
- 战术指令：战前选择
- 文字播报战斗过程

### 5.2 阵型系统

```
前排：承伤，优先被攻击
     步兵、骑兵 适合
中排：稳定输出
     枪兵 适合
后排：远程/法术，伤害高但怕突袭
     弓兵、法师 适合
```

- 前排被击破后，后排直接暴露承伤
- 阵型槽位随营地等级扩展
- 没有前排时，中后排单位承伤 +50%

### 5.3 战术指令

每场战斗前选择一个战术：

| 战术 | 效果 |
|------|------|
| 稳扎稳打 | 防御 +10%，速度 -5% |
| 全军突击 | 攻击 +15%，承伤 +10% |
| 远程压制 | 后排伤害 +12%，前排承伤 +8% |
| 消耗战 | 每回合回复少量生命，战斗回合 +3 |

### 5.4 情报侦察

战斗前可侦察敌方情报：
- 敌方兵种构成
- 敌方属性（高护甲/高魔抗/高血量）
- 敌方弱点提示
- Boss 额外显示特殊技能

示例：
```
敌方：重枪兵方阵 + 少量骑兵
弱点：怕步兵近身，后排无弓兵
建议：前排多放步兵，中排枪兵防骑兵冲锋
```

### 5.5 伤害公式

```
实际伤害 = 攻击 × 技能倍率 × 克制倍率 × 法师易伤系数 × 战术系数 × 随机浮动(0.9~1.1) − 防御减免
```

克制倍率（普通兵种间）：
- 优势克制：1.35
- 普通对阵（含同类型）：1.00
- 劣势被克：0.75

法师易伤系数：
- 法师攻击普通兵：1.3
- 普通兵攻击法师：1.3
- 法师对法师 / 普通兵对普通兵：1.0

> 克制倍率（1.35/1.0/0.75）× 法师易伤（1.3/1.0）= 最大可达 1.35×1.3 ≈ 1.75
> 温和起步，不设 2 倍以上倍率

## 六、军营系统

### 6.1 营地

- 人口容量上限（可升级）
- 建筑列表：采集建筑、兵营、防御设施
- 营地等级影响驻军上限

### 6.2 出战阵容

- 独立于营地的上阵槽位
- 需升级扩容槽位数量
- 包含阵型配置（前中后排）
- 针对敌人情报调整配比

### 6.3 常驻战斗（Phase 3+）

- 周期性 Boss 防御战
- 营地旁固定刷怪点
- 闲置兵力自动刷取功勋

## 七、英雄系统（Phase 3+）

### 7.1 核心定位

```
英雄 = 方阵核心单位 + 技能载体 + 装备成长 + 阵容流派发动机
```

- 英雄占 1 个出战槽位，站在前/中/后排某位置
- 本身是战斗单位，有血量/攻击/防御，不是全局挂件
- **每场战斗只能上阵 1 个英雄**

### 7.2 主动技能

每个英雄带 1~2 个主动技能，战斗中回合触发：

| 技能类型 | 例子 |
|---------|------|
| 指挥类 | "全军冲锋"：本回合所有前排伤害 +20% |
| 伤害类 | "暴风雪"：对敌方后排造成范围伤害 |
| 控场类 | "盾墙"：本回合前排承伤 -30% |
| 治疗类 | "战场包扎"：回复我方血量最低单位 |
| 针对类 | "斩首"：跳过前排直击敌方后排 |

### 7.3 装备系统

英雄有独立装备栏（武器/防具/饰品），影响技能效果和属性。

### 7.4 流派发动机

英雄决定阵容流派方向。同一套兵种，不同英雄打出完全不同效果：

| 英雄类型 | 流派方向 |
|---------|---------|
| 骑兵统帅型 | 骑兵数量越多，冲锋伤害越高 → 骑突流派 |
| 步阵防守型 | 步兵承伤时概率反击 → 铁壁反击流 |
| 弓兵指挥型 | 后排弓兵集火优先击杀 → 狙杀流 |
| 法师增幅型 | 法师易伤倍率从 1.3 提到 1.6 → 玻璃大炮流 |
| 混合型 | 场上每有一种兵种，全体 +5% → 万金油流 |

### 7.5 决策链

```
侦察情报 → 选择英雄(流派方向) → 搭配兵种 → 编排阵型 → 选择战术 → 开战
```

## 八、节奏控制

### 8.1 多重门槛

```
资源足够 + 建筑等级达标 + 击败守关Boss → 解锁下一层
```

### 8.2 建筑升级时长

| 等级 | 耗时 |
|------|------|
| 1→2 | 5 分钟 |
| 3→4 | 30 分钟 |
| 5→6 | 2 小时 |

### 8.3 人口上限

高级兵消耗更多人口，人口靠营地扩张。

### 8.4 等待期可做的事

| 事项 | 收益 |
|------|------|
| 刷低层敌人 | 功勋、经验、掉落 |
| 侦察下一层 | 提前了解敌情 |
| 调试阵容 | 优化配比 |
| 营地防御事件 | 稀有奖励 |
| 养英雄 | 独立成长 |

> 原则：做有收益，不做也不卡死。

## 十二、长期游玩设计

### 12.1 设计原则

游戏不是限时运营型，而是单机永久可玩型。通过以下系统层层叠加，让玩家在单机环境下持续获得成长目标。

```
主线推进 → 远征篇章 → 英雄装备 → 挑战词条 → 周目轮回
   ↑                                              │
   └──────────── 传承加成 ← 新一轮更强 ──────────────┘
```

### 12.2 远征篇章

远征是永久开放内容，不设限时。和主线的区别：

| | 主线 | 远征 |
|------|------|------|
| 敌人 | 固定配置 | 独立敌人 + 地形词条 |
| Boss | 每层 1 个守关 | 独特 Boss 机制 |
| 奖励 | 解锁下一层 | 专属奖励（装备、英雄碎片） |
| 重置 | 不重置 | 可重复挑战 |
| 开放 | 依次解锁 | 达成条件后永久开放 |

每个远征拥有：
- 独立敌人编队（可能混合不常见兵种组合）
- 地形词条（如"山地：骑兵速度 -20%"、"沼泽：步兵防御 -10%"）
- Boss 特殊机制（如"每回合召唤 2 个小兵"、"血量低于 50% 时狂暴"）
- 专属掉落（远征限定装备、英雄解锁碎片）

### 12.3 周目轮回

玩家通关一个大阶段后，可选择"进入新纪元"。

**轮回保留：**
- 英雄（含等级、技能、装备）
- 兵种图鉴（已解锁的兵种永久记录）
- 永久天赋/传承加成
- 部分特殊装备

**轮回重置：**
- 资源清零
- 建筑等级重置
- 普通兵种数量清零

**传承点：**
- 根据轮回前进度结算传承点
- 传承点用于解锁永久加成（资源产出 +X%、兵种属性 +X% 等）
- 多周目累加，越轮回越强

### 12.4 挑战词条

玩家在战斗/远征中可主动选择额外难度词条，挑战与奖励成正比。

**词条示例：**

| 类型 | 词条 | 效果 | 奖励加成 |
|------|------|------|----------|
| 敌人强化 | 狂化 | 敌方攻击 +20% | +15% |
| 敌人强化 | 钢铁之躯 | 敌方防御 +30% | +15% |
| 我方限制 | 物资紧缺 | 出战兵力上限 -2 | +20% |

### 12.5 资源回收（长期消耗口）

避免后期低级资源无用，每个层级资源都有持续消耗出口：

| 资源 | 消耗出口 |
|------|---------|
| T1 木材/石料/食物 | 兵种训练消耗、营地维护、远征补给 |
| T2 铁矿/煤炭 | 高级兵维护费、装备锻造材料、建筑加速 |
| T3 秘银/水晶 | 英雄装备强化、神级单位前置、传承解锁 |
| T4 龙晶/神血 | 神级兵种训练、周目轮回保留道具、终极装备 |

核心原则：
- **低级资源永不过期**：T1 到后期仍然需要大量消耗
- **资源可向上转换**：比例递减（如 100 木材 → 1 铁矿）
- **高级资源稀缺**：T3/T4 无法靠低级转换大量获得

### 军粮维护费

所有士兵持续消耗食物，净产出 = 食物采集 − 维护费 − 人口口粮(0.1/人)。净负且库存归零时食物枯竭，变相限制军队规模。

### 训练队列

`S.queue[unit] = {count, timer, reason}` — 每 tick 生产1人，队列上限=后备×10。资源不足或后备满时暂停并显示原因。
